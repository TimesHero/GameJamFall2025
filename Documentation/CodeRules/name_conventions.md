* Code Rules: Naming Conventions

  Naming conventions for variou elements of code

  - **Variable Names:**
      * Snake Case
        + **Example:** variable_name
      * Add "m" before private variable/member
          + **Example:** private m_variable_name

  - **Function Names:**
      * Camel Case
          + **Example:** functionName

  - **Class Name:**
      * Pascal Case
          + **Example:** ClassName

  - **File Names:**
      * Snake Case


  - **Directory Names:**
      * Pascal Case
